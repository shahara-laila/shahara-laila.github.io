## What I Learned

This course covered the fundamentals of software quality assurance — from the Software Development Life Cycle (SDLC) models to systematic testing strategies. It grounded my understanding of how professional software teams ensure reliability before and after deployment.

## Key Topics Covered

- **SDLC models** — Waterfall, Spiral, V-model, Agile, their trade-offs and when to use each
- **Testing fundamentals** — definitions, objectives, principles, and psychology of testing
- **Testing levels** — unit, integration, system, and acceptance testing
- **Test types** — functional, non-functional, regression, smoke, and exploratory testing
- **Test documentation** — test plans, test cases, test reports, bug life cycle
- **Strategic approaches** — black-box, white-box, gray-box testing techniques

## Why It Matters to My Work

Machine learning models need rigorous evaluation — both as software systems and as scientific tools. Understanding testing frameworks helps me design better validation pipelines for my ML models and ensures that the web platforms I build (like the Oral Carcinoma Detection Platform) are reliable before clinical use. As an aspiring academic, these skills also inform how I teach software engineering principles.

## Key Takeaways

Testing is not an afterthought — it is a design discipline. Systematic testing strategies, applied to both code and ML model evaluation pipelines, are what separate research-grade work from production-ready systems.
