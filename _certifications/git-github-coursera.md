## What I Learned

This course provided a comprehensive, hands-on journey through Git and GitHub — from foundational version control concepts to advanced team collaboration workflows. Having used Git informally, this course filled critical gaps in my understanding of branching strategies, rebasing, and repository management at scale.

## Key Topics Covered

- **Version control fundamentals** — understanding commits, staging, and history
- **Branching and merging** — feature branches, merge conflicts, fast-forward vs. 3-way merges
- **Remote repositories** — working with GitHub, push/pull, fork and clone workflows
- **Collaboration** — pull requests, code review workflows, issue tracking
- **Advanced Git** — rebasing, cherry-picking, stashing, tagging
- **GitHub features** — Actions overview, Pages, project boards

## Why It Matters to My Work

As a researcher working on code-heavy projects (YOLO models, Django platforms, LangChain pipelines), clean version control is essential. This certification strengthened my ability to manage research codebases, collaborate cleanly, and maintain reproducible project histories — which is critical when submitting code alongside academic papers.

## Key Takeaways

Proper Git discipline — meaningful commit messages, atomic commits, and clear branching — makes research reproducible and collaborative. I now apply these practices consistently across all my projects.
